# CLAUDE.md — Горький.Акселератор Landing Page

## Проект

Одностраничный лэндинг для акселерационной программы «Горький.Акселератор» (Нижний Новгород).
Скелет уже сверстан (`gorky-accelerator.html`). Задача — интегрировать дизайнерские ассеты и привести вёрстку в соответствие с утверждённым дизайном.

## Технологии

- Чистый HTML/CSS/JS (single-file, без фреймворков)
- SVG-ассеты инлайнятся или подключаются через `<img>` / CSS `background-image`
- PNG-изображения подключаются через `<img>`
- Шрифты: Google Fonts — `Unbounded` (display) + `Onest` (body)
- Адаптивность: мобильные (≤480), планшеты (≤768), десктоп (≤1024), wide (>1024)

## Ключевые правила вёрстки

1. **Hover-эффект** — на ВСЕ карточки и интерактивные блоки: лёгкая подсветка нейтральным цветом (`rgba(6,170,189,0.06)` для светлых, `rgba(6,170,189,0.08)` для тёмных), плавный transition 0.35s.

2. **SVG-ассеты — НЕ инлайнить содержимое base64-картинок.** Файлы `фото_1-4.svg`, `рисунок.svg` содержат embedded base64-изображения. Подключать через `<img src="assets/svg/фото_1.svg">`.

3. **Два регистра `подложка.svg`** — lowercase (539×614, белая, для карточек «преимущества»/«для кого») и uppercase `Подложка.svg` (394×223, тёмная, для карточек «программа»). Это РАЗНЫЕ файлы.

4. **Таймлайн** — по дизайну ГОРИЗОНТАЛЬНЫЙ (не вертикальный как в скелете). Линия идёт слева направо: Фев → [точка] → Март → [точка] → Май. Используй `сетка.svg` как скелет + `отметки_1.svg`/`отметки_2.svg`. На мобильных — вертикальный fallback.

5. **Карточки «Преимущества»** (стр. 2 PDF) — 4 карточки в ряд, белые/полупрозрачные, заголовок голубой (#06aabd), текст тёмный, внизу каждой — SVG-иллюстрация (`фото_1-4.svg`).

6. **Секция «Программа»** (стр. 4 PDF) — тёмный фон, карточки в асимметричной сетке: 3 сверху + 2 снизу сдвинуто вправо. Внизу слева — декоративный `рисунок.svg`. Каждая карточка = `Подложка.svg` как бэкграунд + текст + иконка-стрелка.

7. **Секция «Для кого»** (стр. 5 PDF) — градиентный фон (бирюза→белый), 3 белые карточки с `подложка.svg` как форма, иконка-стрелка в правом верхнем углу. Декоративный 3D-торус (`hero-3d-2.png`) в верхнем левом углу.

8. **Hero** — НЕ использовать canvas-частицы (убрать из скелета). Фон = тёмный градиент (#003366 → #06aabd диагонально). Справа сверху — белое свечение. Заголовок «ГОРЬКИЙ.\nАКСЕЛЕРАТОР» огромным шрифтом (как в `название.svg`). Кнопка — белая с чёрным текстом и стрелкой (как в `кнопка.svg`).

9. **Текст описания Hero**: «Практико-ориентированная образовательная программа с экспертной поддержкой для развития управленческих, продуктовых и коммерческих компетенций ИТ-компаний, с фокусом на рост, инвестиции и пилотирование.»

## Порядок работы

1. Скопировать `gorky-accelerator.html` → работать над копией
2. Создать структуру папок `assets/svg/` и `assets/img/`
3. Скопировать все ассеты в соответствующие папки
4. Пройтись БЛОК ЗА БЛОКОМ, сверяясь с описанием в `ASSETS.md`
5. Протестировать адаптивность (мысленно проверить breakpoints)
6. Убедиться, что hover-эффекты работают на всех карточках
